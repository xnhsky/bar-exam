## 第0項 設計の三原則

### 原理1 解説の対象は記述であって、肢ではない（第一原理）

肢（①アエ等の組合せ記号）は出題の包装。学習資産は記述ア〜オの法理だけ。
- 「なぜ①アエが正解か」のナラティブ導出を書かない。判定は PART A の reveal（記述○×一覧表）が
  構造として持つ。
- PART B は出題形式によらず記述（ア〜オ／単純5択は肢1〜5）単位。
- 選択戦略語彙（消去法・絞り込み・2肢で正解 等）は全域禁止（導き書§3-1 と正反対の有害情報）。

### 原理2 記述（肢）が管理単位（v0.4 中核・Lexia 連携）

過去問の各記述を、Lexia の **弱点記録と再復習の単位**にする。
- **解く単位は問題のまま**（5記述まとめて解く＝導き書§3-1）。画面に出るのは問題1枚。
- **追跡・再復習の単位は記述**。解答時に各記述の○×を収集し、
  - **間違えた記述だけ**を要復習プールへ（肢キー `{問題ID}#stmt-{記述}`）。**正解した記述は
    プールに入れない**（既存プールの「不正解時のみ追加」を肢粒度に踏襲）。
  - 間違えた記述を弱点アーカイブ（mistake_archive）へ肢キーで記録 → 弱点克服帳・試験日
    シミュレーションが**記述/論点粒度**になる。
  - プール内の記述は、別日に正解を重ねれば卒業（既存の累積2回正解ルール）。
- これにより導き書§3-6 の理想（評価1〜2で「間違えたその肢にすぐ再会」）が**初めて実装**される。

### 原理3 12問クイズ（PART D）は廃止

PART D は本番設計から削除する。理由：
- 本物の記述（約4,500問×5＝約2.2万肢）が、各々別の事案で同じ論点を問う。合成12問ドリルより
  はるかに多彩な「論点を新事案で見抜く」反復になる。
- 論点の規範暗記は RX 論証カードが担当（規範を隠して想起）。12問は TX肢 と RX の中間で役割重複。
- 廃止で **TX肢＝問題文→論点の認識＋○×精度 ／ RX＝規範の想起** の2層に整理される。
- 生成コスト最大の部品が消え、core/deep 双方が軽くなる（導き書の「12問＝＋300〜450時間」も不要）。

### 原理4 周回コアと深掘り別冊の分離（データ駆動深掘り）

深掘り層は全問事前生成せず、肢/論点の弱点データが解禁条件を満たした問題にだけ後追い生成する
（導き書 鉄則②）。コアは生成後 immutable。

### 三層モデルとの対応

| 導き書の層 | v11 での実体 | 使う場面 |
|---|---|---|
| 周回層 | PART A（5記述に○×→reveal→一行解説）。○×は Lexia が肢キーで取得 | 毎周 |
| 誤答修正層 | 間違えた記述だけが要復習プールで再来＋PART B 該当記述＋参考条文判例＋体系/放射マップ | 誤答時のみ |
| 深掘り層 | 別冊 `-deep.html`（教授③④・判例完全プロファイル・フローチャート・PART C）。**PART D なし** | 解禁条件成立時と直前期のみ |

## 第2項 コア（メイン）の構成

```
HEAD（:root 配色 V3）
HEADER（exam-meta・配色記載禁止 G8）
PART A ── 問題情報
  ├ problem-text（問題文＋記述ア〜オ原文）
  ├ answer-area（data-answer-type="ox-grid"・5記述の○×を1つずつ収集）
  │   ・各記述に ○/× ボタン（ox-row）。reveal で記述ごとに採点。
  │   ・data-correct-value＝記述別正誤の連結（例 "××○×○"）
  │   ・final-answer に【記述○×一覧表】（G23）。表は機械可読：
  │       <table data-answer-key="ア:x,イ:x,ウ:o,エ:x,オ:o">
  │       各行 <tr data-stmt="ア" data-verdict="x">。論点コアのセルを含む。
  │   ・この○×が Lexia の肢キー記録の一次情報源（原理2）
  └ data-explanation（記述単位の一行解説・組合せ導出を書かない）
PART B ── 記述別解説（記述単位・5セクション）
  順：verdict → choice-points（論点コア・前倒し）→ 記述原文 → 解説原文（法理→ズレ）
      → 根拠リンク（参考条文判例へ）→ 教授①②（③④は別冊）
参考条文・判例 ── 条文（文言＋保護法益＋制度趣旨）／判例（重要度濃淡・規範＋射程要点）
      ※完全プロファイル【事案】【判旨原文】【補足】は別冊 D-2
体系的マインドマップ ── 体系ツリー（mindmap-tree）＋放射マップ（mindmap-radial）の2枚
footer-spec（feature-tag 先頭 = TX v11.0.0 LOOP-CORE・別冊リンク）
```

- 別冊専用：教授③④・判例完全プロファイル・フローチャート（flow-svg）・PART C。**PART D は存在しない**。
- SVG はコアに2枚（ツリー＋放射）。AABB 衝突検査は両図に実施。1メッセージ50KB超 Write/Edit 禁止。

## 第3項 choice-points の内容規律（論点コア原則）

各バレットの主語は法概念（罪名・要件・判例法理・概念）に限定。
- ✅ 規範のコア／判例の結論と射程／類似概念の区別基準／決め手の限定句
- ✅ 任意で1点「ひっかけの型」（既存 `exam-mark` マーカー流用）
- ❌ 「正解は肢N」／組合せ判定／「本記述は誤り・正しい」／他記述参照

## 第4項 禁止句（機械検証）

```
正解は肢 / 組合せ全体 / 組合せ不成立 / 混在（組合せ文脈）
消去法 / 絞り込み / 〜が分かれば正解 / 〜を切れれば / 判断できれば肢
choice-points 内：本記述は誤り / 本記述は正しい / 記述[ア-オ]は（他記述参照）
```
AP-30〜40（ネタバレ防止）は PART A について継承。
