## 第1-bis項 過去問原文の不可侵ブロック（§v13n・事例型のみ・2026-07-10 合意）

共有の事例/見解を持つ事例型（「甲の罪責」型・見解×事例型）では、`#part-a:has(.tx-inline-card) >
.problem-text{display:none}` が事例を隠して「事例が消える」事故になる（刑TX374 で実害）。
**別エリア方式**で解決する。詳細規約の正典は `docs/tx-v12.2.1-inline-lock.md` §v13n。

1. **不可侵原文ブロック `.tx-original-block`**（問題文直下・解法ナビ/肢カードより上）＝過去問PDFの
   問い＋事例＋（見解）＋記述/罪名列を**逐語のまま常時表示**する箱。付け足し・要約・改変ゼロ
   （区画ラベル「📖過去問原文」「【事例】/【罪名】」のみ可）。事例は `.case-description > .case-scene >
   .case-paragraph` 構造（Lexia `extractCaseContext` が各設問へ文脈添付）。CSS は canonical/GENESIS-CARD.html
   同梱（ブロックが無ければ不活性）＝複製で自動継承。
2. **型による使い分け**：共有の事例/見解を持つ型→ブロック**要**／各記述が独立自己完結の型
   （逐語のインラインカードがそのまま原文）→**不要**（作ると長文の二重掲載になるだけ）。
3. **一問一答の数＝実質の記述数**（表面の選択肢記号数ではない。組合せ→素材の記述、空欄→空欄数）。
4. **恒久ゲート**：**G61**（丸囲みマーカー/ラベルの `text-indent:0`）・**G62**（`.ox-row` 数＝`.tx-charge` 数
   ＝answer-key 長）。いずれもブロック無しはスキップ（誤爆ゼロ）。run() と push 前ゲート
   `check-tx-lex-engine.py` の双方で実行（2026-07-11 監査で独立ゲート化）。

  一部＝`is-partial`「n / N 正解」／`answered==0` は非表示）。行の正誤は `.tx-user-correct`/`.tx-user-incorrect` で判定

  text-indent:0（パネル内 `<p>` が継承するグローバル字下げを打ち消す）。CSS は canonical 同梱。

4. `.sub-card.basis-link` … 📚 BASIS（第6項）。
5. `.tx-v13-trap` … ⚠️間違いやすいポイント（**食み出しタブ**＋本文1字下げ）。**横串・誤解の罠を積極投入**

## 第5-bis項 Lexia 復習プール（SM2）への配線 ★v13 で必ず更新する

Lexia は `_lex` を取り込み、間違えた記述を **肢キー `{問題ID}#stmt-{記述}`** で SM2 追跡し、一問一答の
復習カードとして提示する。Lexia が復習カードに使う要素（`Lexia/src/App.jsx`）:

- **正解＝ `[data-answer-key]`**（正誤表 `.statement-verdict-table` の `data-answer-key`。権威・単一情報源）。
- **記述文＝ `.ox-row .ox-stmt`**（短ラベル）→ 短い場合 `.syn-orig` にフォールバック。
- **解説（最優先）＝ `.ox-row .ox-pool-explain`**：`.ox-pool-gist`（要点1文）＋ `.ox-pool-points li`
  （`ラベル：本文` を Lexia が `/^([^：:]{1,30})[：:](.*)$/` で分解）。
- **物語＝ `.fa-narrative`**（プールカードにストーリーを載せ、該当段落を強調）。

**v13 配線ルール（LOCKED）**：`.ox-pool-explain` は **v13 のカード内容と同期させる**。
- `.ox-pool-gist` ← 各カードの **💡THE GIST（`.syn-lead` の本文）**（○×結論を含む要点1文）。
- `.ox-pool-points` ← 各カードの **📌POINT（`.choice-points` の各 `li`）**（`実体ラベル：本文`。ひっかけの型を含む）。
- **旧 v12 の5点フロー（文言/趣旨/射程/切断点/転用）を `.ox-pool-points` に残さない**（v13 では廃止した内容で、
  劣化コピーの温床。実体ラベルへ置換＝記号フリー・G30/G50）。
- `.ox-pool-explain` の `data-source` は v13 由来と分かる値にする。
- **正誤表 `.statement-verdict-table`（answer-key 源）と `.answer-ox-grid`／`.ox-row[data-stmt]` は DOM 保持**
  （inline-prototype-mode で非表示でも、Lexia/SM2・answer-key・検証の単一情報源として消さない）。

> v13 化・生成では、カード解説（synthesis/POINT）を書いたら **必ず同じ内容で `.ox-pool-explain` を更新**する
> （伝播ツール例＝scratchpad build_v14c：THE GIST→gist、POINT→points を機械再配線）。

### 第5-bis-2項 ox-stmt の要件事実完全性 ★2026-07-13（G65）

Lexia は `.ox-stmt` が **30 字以上なら「自己完結命題」とみなし、記述原文（`.syn-orig`）への
フォールバック無しで単独の一問一答カードとして出題**する（Lexia App.jsx・30 字未満のみ原文へ
フォールバック）。したがって ox-stmt への圧縮は「短くする」作業ではなく、**原文を知らない読者が
その一文だけで○×を判定できる形に書き換える**作業である。

- **省略禁止（正誤の分かれ目になる要件事実）**：
  1. **主体の身分**：公務員／医師／弁護士 等（身分犯・真正身分犯の前提。156条・160条の主体等）
  2. **目的要件**：行使の目的・営利の目的 等（目的犯の成立前提）
  3. **客体の性質**：公文書か私文書か（市立/県立/公立/私立 等の官公性・「公文書である〜」）
  4. **権限・委託・承諾の有無**：作成権限の有無・名義人の承諾・無断 等
- **規範名の主語化**：規範命題型の ox-stmt（「〜まで要しない」等）は、何罪のどの文言の話かを
  冒頭に残す（例：「職権行使の相手方の…」→「公務員職権濫用罪の成立には、職権行使の相手方の…」）。
  罪名プレフィックスを削ると設問の帰属先が消える。
- **自己照合（執筆者本人・1回）**：ox-stmt を書いたら「原文を知らない法律家がこの一文だけで
  ○×を付けられるか」を1回自問する（一見自己完結テスト）。判定に必要な事実が落ちていれば
  原文から復元する（CLAUDE.md §3 省エネ規律 (b) の一項目）。
- **機械ゲート**：`validate-tx-core.py` **G65**（WARNING・語彙差分ヒューリスティック）が
  「原文（inline カード提示文／syn-orig）にあるが ox-stmt に無い」重要語（公務員・行使の目的・
  医師・市立/公立系・承諾 等）を検出する。push 前 `check-tx-lex-engine.py` にも非ブロッキング
  助言として常設。言い換え（例：市立→「公務員である」・承諾なく→「無断で」）で事実が保存されて
  いる場合の誤検出は無視してよいが、無視する前に必ず原文と1回照合する。
- **実害（この規約の由来）**：刑TX378 記述3＝原文「X市立病院の事務長を務める公務員…行使の目的で…」
  が ox-stmt「病院事務長が…」へ圧縮され、虚偽公文書作成罪(156)の真正身分犯性（公務員か私人か・
  公文書か私文書か）が復習カード単体で判定不能になった。2026-07-13 に同型 32 記述／18 ファイル
  （刑TX063/073/077/181/201/261/370/371/375/378/379/380/384/386/420/422/423/426）を一括是正。
- **断定命題形式（★2026-07-18・G70）**：ox-stmt は**要件事実込みの断定完結命題**（文末は句点）で書く。

  禁止。設問の枠組み（「〜という見解の根拠となり得るか」・本文の結論を前提とした検討 等）に判定対象が
  置かれている問題は、その枠組みを各 ox-stmt の文内に取り込んで自己完結させる（例：「〜は、犯人が他人を
  教唆して自己を隠避させた場合に犯人隠避教唆罪の成立を認める見解の根拠となり得る。」）。共有事例型は

  語彙差分では構造的に検出できない＝2026-07-13 是正の漏れ経路）。「〜との記述」「〜との批判が可能で
  ある」等の判定枠が読める定型は可。**ただし「〜との内容/とする内容」は判定枠にならない**＝設問枠組み
