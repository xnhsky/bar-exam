## v12 の主導線

1. 問題文の各肢直後に `.tx-inline-card[data-stmt]` を置く。
2. 各カードは、肢本文、○×ボタン、条文原文カード、判断ブロック、記憶フック、答案圧縮、詳説トグルを持つ。
3. 裏の `.answer-area[data-answer-type="ox-grid"]` と `.answer-ox-grid` は削除しない。Lexia/SM2 記録、answer-key、検証の単一情報源として保持し、インライン面から同期する。
4. 「解説だけ閲覧」は `answered` を付けず、SM2 記録を発生させない。
5. `reveal` 時は従来の最終解説を残しつつ、解説冒頭の正誤表にユーザー回答列を追加して比較表示する。

## 各肢カードの標準順序

```
article.tx-inline-card[data-stmt]
  .tx-inline-head
    .tx-inline-stmt
    .tx-inline-actions (.tx-inline-ox ○ / ×)
  .tx-inline-explain
    .tx-answer-box           # ① 答案圧縮＝ANSWER 箱（問題文直下・英語食み出しタブ ✍ ANSWER）
    .tx-mini-law
      .tx-law-item.is-main     # 主条文。ブルー系（濃）。
      .tx-law-item.is-support  # 補助条文。ブルー系（中）。
      .tx-law-item.is-context  # 文脈条文。ブルー系（淡）。
      .tx-law-item.is-case     # 判例。ピンク系（必ず is-case）。
    .tx-article-flow
      文言 / 趣旨 / 射程 / 切断点 / 転用   # ラベルは楕円ピル＋立体（重厚感）
      .tx-onepoint             # ② 記憶フック＝💡 ワンポイント（解説末尾）
      details.tx-inline-detail # ③ 詳説。必ず .tx-detail-panel[data-partb-source="N"]（N=data-stmt）。空 details 禁止＝G43。
```

> **v12.2.0 で物理順序を変更**：旧 `.tx-cycle-aids`（記憶フック＋答案圧縮を末尾2カラム）は廃止。
> 答案圧縮→冒頭 `.tx-answer-box`、記憶フック→末尾 `.tx-onepoint`。`scripts/tx-lex-answerbox.py` が
> 既存物を冪等に移設し、空 `<details>` には `data-partb-source` panel を補完する。

## 条文・判例カード

- 条文番号は「刑法」「第百八条」のように、法令名と条番号を別ラベルにする。
- 条文本文は原文を主役にする。注釈は条文カード内で目立たせず、下の判断ブロックまたは詳説へ寄せる。
- 主条文はブルー系、判例はピンク系、補助条文・文脈条文はグレー系で、重要度の濃淡を視覚化する。
- 試験慣れのため、条文番号は漢数字表記を標準とする。
- 上段には、下部の参考条文・判例 `basis-card-header` から取得した題名チップと、法理テーマチップを置く。ただし `題名：` / `テーマ：` という説明文字は表示しない。
- 本文冒頭の `本条` / `本文` / `①` / `②` / `判旨核心` / `判例` は `.tx-mini-law-para` で小ラベル化し、本文は `.tx-mini-law-body` に入れる。2項以上の条文だけ項番号ラベルを出す。
- ラベル付き本文は2カラムにし、段落本文の先頭だけ1字下げる。折返し行をラベル下へぶら下げない。判例本文も同じ規律に従う。

## 答案圧縮（ANSWER 箱）・記憶フック（ワンポイント）　※v12.2.0 で配置変更

- **答案圧縮＝解説冒頭（問題文直下）の `.tx-answer-box`**。英語食み出しタブ `✍ ANSWER`（CSS `::before` 固定）。
  「要件/条文 → 本件 → 結論」の形で、論証や短文答案へ転用できるテーゼにする。解説は既定 hidden なので、
  冒頭に置いても周回前のネタバレにはならない（○×後の確認で最初に結論の型を見せる導線）。
- **記憶フック＝解説末尾の `.tx-onepoint`**（💡 ラベル・CSS `::before` 固定）。一文のワンポイント。
- どちらも周回出口の一文であり、正解番号だけを示すネタバレ文にしない。
- 旧 `.tx-cycle-aids`（紫＝記憶フック／ピンク＝答案圧縮の末尾2カラム）は v12.2.0 で廃止。

## 解説品質レビュー（v12.2.1）

- 解説内容は TX の品質の中核であり、機械整形・一括置換・テンプレ埋めだけで確定してはならない。
- 新規生成・既存更新・一括伝播後は、作業時点で利用可能な最上位モデルを使い、推論/エフォート設定を最大にして内容レビューを行う。
- 機械検査は構造・表示・欠落・旧UI残存の補助であり、PASS しても解説内容の正確性を保証しない。
- レビュー対象は、`ANSWER`、条文/判例ボックス、5点フロー、コツ、記憶フック、詳説、物語解説、SM2 用 `.ox-pool-explain`。
- 問題文事実、条文番号・項、判例名・判旨、正誤結論、文言/趣旨/射程/切断点/転用の対応を個別照合する。
- 内容レビュー未了のファイルは、v12.2.1 完了・配信可・push可とは扱わない。

## 最新法令・判例・学説レビュー（v12.2.1）

- 生成・更新時点で、最新の条文、判例、主要学説を確認する。法令は e-Gov 法令検索、司法試験の法令基準は法務省の当該年度実施情報/Q&A/試験用法文、判例は裁判所裁判例検索と最近の裁判例を一次確認先にする。
- 学説は最新版の基本書、判例百選、重要判例解説、法学教室・法学セミナー等で補完する。機械検索だけで有力説の現状を確定しない。
- 新旧差分がある場合は、本文を現在法・最新判例ベースへ更新し、旧法・旧判例・旧説は `tx-current-law-note` コラムで補足する。
- `tx-current-law-note` には、確認日、参照ソース、旧処理、現在の処理、立法経緯・改正経緯、改正趣旨、試験対策上の扱いを記載する。
- `tx-current-law-note` は見出しバッジ＋2カラム本文とし、`.tx-law-note-label` は左列、`.tx-law-note-body` は右列に置く。本文列だけ `text-indent:1em` を維持し、ラベル横に長文を直置きしない。
- 最新確認未了のファイルは、内容レビュー未了と同じく、v12.2.1 完了・配信可・push可とは扱わない。

## 物語解説のタイポグラフィ

- `.fa-narrative` は初回理解のための読み物であり、周回時の主導線ではない。
- 本文は通常ウェイトを基本とし、重要語だけを `<b>` で示す。
- `<b>` は色で目を止める。太さで押し切らない。正典値は `.fa-narrative b{ font-weight:560; }`。
- iPhone では濃い明朝太字が潰れて見えるため、`600` 超や `700` 系の太字へ戻さない。
- inline 型では reveal / browse 前に物語解説を表示しない。表示後の `data-fa-label` は通常フロー内の左寄せラベルにし、絶対配置で本文に被せない。
- `data-fa-label` 段落の本文は `.fa-narrative-body` で包み、本文先頭だけ `text-indent:1em` にする。ラベル・見出し・バッジには字下げを掛けない。

## 回答後フィードバック

- `解答を表示` 後は、回答操作パネルに「あなたの答え」「正解」「相違」を表示する。
- ○×グリッドでは、相違を `記述N（あなた ○／正解 ×）` で表示し、選択ミスの箇所を視認できるようにする。
- 問題カード内の判定欄は、○×ボタンの下に2行で置き、ユーザー選択と正解を `あなた：×` / `正解：○ 正しい` のように表示する。
- 解説冒頭には、正誤表を常時表示し、`あなた` 列を追加して比較できるようにする。正解行は緑背景、誤答行はピンク背景、未回答は淡色で表示する。
- トーストは廃止する。回答後フィードバックは、解説冒頭の正誤表とカード内2行判定に集約する。
- 記憶フック本文 `.tx-onepoint .tx-op-body` は本文先頭だけ `text-indent:1em` にする。ラベルや本文列全体の余白では代用しない。

## 問題都合ラベルの扱い

- 記号問題、空欄問題、見解ラベル問題では、`.ox-stmt`、`.tx-reflex-core`、`.tx-cycle-aids`、`.ox-pool-explain` に問題都合のラベルを残さない。
- Lexia/SM2 の表面に入れるのは、組合せ番号や空欄記号ではなく、論点コア・テーゼ・自己完結命題である。
- 公式問題文の原文は保持する。置換するのは学習資産側だけである。

## Lexia/SM2

- 復習プールの最優先ソースは `.ox-row .ox-pool-explain`。
- v12 inline 型では、`.tx-reflex-core` と `.tx-cycle-aids` の要点を `.ox-pool-explain` にミラーする。
- narrative、詳説本文、問題固有ラベルは SM2 payload に混ぜない。復習プールは「次回それだけを見て○×判断を復元できる最小テーゼ」にする。

## 問題型タクソノミーと ox-grid 変換規律（v12.1.1・2026-06-30）

短答の出題形式は多様だが、`_lex` の学習資産（ox-grid／ox-stmt／論点コア／SM2 プール）は
**型に依らず「単独で○×判定できる最小学習単位」へ分解**する。以下を正典規律とする。

### 問題型タクソノミーと _lex マッピング

| 型 | 公式 000_TX | _lex ox-row（行）の単位 | correct-value | 見本 |
|---|---|---|---|---|
| 単純○×／独立5択 | `single` | 各記述（肢1〜5）独立○× | 各記述の真偽 | 既存多数 |
| 記述ア〜オ組合せ | `single`（組合せ番号） | **各記述ア〜オを独立○×** | 各記述の真偽 | 既存多数 |
| 空欄補充（①〜に語句を当てる） | `single`（組合せ番号） | **各空欄**（`data-oxgrid-mode="blank"`） | **全○** | **刑TX350** |
| 見解／学説（A説・甲乙） | 出題形式に従う | 各記述／各空欄（見解は実体の学説名で主語化） | 同上 | 刑TX351 |
| 事例（事例Ⅰ/Ⅱ） | 出題形式に従う | **各事例の法的論点**（事例ラベルは事案の実体内容へ置換） | 同上 | — |
| 個数（正しいものはいくつ） | `single` | 各記述を独立○×（**個数自体は学習対象にしない**） | 各記述の真偽 | 既存37問 |
| 部分組合せ | 上記いずれか | 独立単位へ分解 | 同上 | — |

### R1（最重要）ox-row／ox-stmt は最小学習単位にする

- **ox-row／ox-stmt は「単独で○×判定できる最小学習単位」（空欄／記述／事例／論点）にする。**
- **答え選択肢（組合せ1〜5）を ox-row にしてはならない。**
- **ox-stmt を「…とする組合せは正しい／誤り」で終える「組合せ当否判定」を禁止**（＝正解番号の暗記で転用不能）。
- correct-value はその単位の○×であり、「正しい組合せを1つだけ選ぶ5択を○×化した **○1個**」構成にしない
  （空欄分解型は原則**全○**、記述独立型は各記述の真偽）。
- **良い例＝刑TX350**（blank モード・各空欄を独立命題化・correct-value 全○）。
- **悪い例（是正対象7問）＝刑TX089／174／212／218／220／256／368**（組合せ当否判定。`validate-tx-core.py` G42 が検出）。

### R2 サブタイプ別の分解先

- **空欄補充**（例 刑TX218・368）→ `data-oxgrid-mode="blank"`、ox-row＝各空欄、ox-stmt＝その空欄に入る内容の法的命題、correct-value＝全○。
- **記述ア〜オ／a〜e 組合せ**（例 刑TX256・174・212・220）→ 標準 ox-grid、ox-row＝各記述、ox-stmt＝その記述単独の真偽命題。
- **事例**（例 刑TX089）→ ox-row＝各事例の法的論点、事例Ⅰ/Ⅱ は事案の実体内容へ置換。

### blank モード（`data-oxgrid-mode="blank"`）

空欄補充組合せ専用。各 ox-row＝空欄、各 ox-stmt＝その空欄の正しい内容を述べる独立命題、
correct-value＝全○、組合せ番号は学習対象にしない。`data-explanation` で「一番堅く決まる空欄から
順に確定し、組合せ番号は結果にすぎない」旨を明示する（見本 刑TX350）。

### 整合ガード G42（組合せ当否判定の禁止）

`validate-tx-core.py` の **G42**：ox-grid の ox-stmt が `組合せ(は|として|を|が)?…(正しい|誤り)` に
マッチ → **ERROR**（correct-value の○数も併記）。`check-tx-lex-engine.py`（push 前ゲート）に常駐。
G30（記号のみ肢）が見逃す「記号を消しても組合せ全体の当否を問う」構造欠陥を捕捉する。

### 問題都合ラベルの外延（検証器と統一）

SM2／論点コアから除去すべき「問題都合ラベル」の外延は `validate-tx-core.py` の
`POOL_LABEL_PATTERNS` と一致させる：記号（①〜⑩・(a)〜(j)）・見解（A説／甲乙説／第N説／Aの見解）・
**事例Ⅰ/Ⅱ**・学生A〜・**「本問」「上記見解」依存**・組合せ番号・記述／肢への相互参照。これらは
実体（学説名・語句・要件・判例テーゼ・事案内容）へ置換する。

### answer-key／肢キーのラベル正規化

`data-answer-key`・`data-stmt` の記述ラベルは**数字（`1:x,2:o,…`）を正典標準**とする
（コーパス多数派・`data-stmt` も数字）。Lexia 肢キーは `問題ID#stmt-{数字}`。
**既存のカナ（`ア:x`）表記は SM2 履歴保護のため据え置き**、新規生成・是正は数字で揃える。

### 概念の境界（論点コア／テーゼ／自己完結命題）

- **論点コア**：条文・判例を **文言・趣旨・射程・切断点・転用**の5点に分解した知識核（`.tx-reflex-core`・G33）。
- **テーゼ**：「要件／条文 → 本件 → 結論」の形で論証・短答に転用できる**一文の到達形**（答案圧縮の出口）。
- **自己完結命題**：カード単独で○×を復元できる文（`.ox-stmt`）。
- 関係：論点コア（分解された知識）→ テーゼ（転用可能な一文）→ 自己完結命題（○×判定可能な出題形）。

### SM2 payload 契約の正典

SM2 payload の単一情報源は **twotrack（`.ox-pool-explain`／`.tx-reflex-core`／G34）世代を正典**とする。
`spec/tx-v11.0.0-core.md` 第9項（`.ox-stmt`／`extractStmtMeta` 世代）は本節で上書き済み。
