## 2-bis. 短答反射の5点セット（TX-LEX 正典ルール）

TX-LEX は「正解根拠を読む教材」ではなく、Lexia/SM2 で回すための **誤肢切断・条文判例反射装置** とする。`.ox-stmt` と verdict「論点のコア」は、過去問の肢を次回以降の類似肢に転用できる形で圧縮する。

各肢のコアには、必要に応じて次の5点を入れる。全部を長く書くのではなく、**1肢1ストーリー／1肢1切断点**でまとめる。

| 要素 | 役割 | 書き方 |
|---|---|---|
| **文言** | 条文のどの語から出発するか | 「○条の『〜』は…」 |
| **趣旨** | なぜその読み方になるか | 制度趣旨・保護法益を一文で置く |
| **射程** | 判例・通説がどこまで言っているか | 事案差・要件差を短く示す |
| **切断点** | 誤肢ならどの語で落とすか | 「危ない語：常に／直ちに／当然に／一切」などを明示 |
| **転用** | 次の類似肢でどこを見るか | 「次は○○の有無を見る」と書く |

- 誤りの肢では、必ず「危ない語」または「過剰一般化」を特定する。
- 正しい肢では、切断点の代わりに「肯定条件」「成立条件」を明示する。
- 解法テクニック（組合せ消去・逆算・選択肢番号の処理）は、`.ox-stmt` に混ぜず、解法ナビに置く。
- 短答主力教材として、説明は長文化しない。移動中に1画面で想起できる密度を上限とする。
- 「判例によれば正しい／誤り」で終わらせない。必ず、条文文言・保護法益・判例射程のどれで正誤が決まるかを残す。

悪い例：

> 判例に照らすと誤り。

良い例：

> **文言**：○条は「〜」を要件にする。**趣旨**：保護法益は〜だから、単なる〜だけでは足りない。**切断点**：「当然に」が過剰。**転用**：次は「〜の認識／危険の現実化」の有無を見る。

## 2-ter. TX-LEX 標準構成（必須・条件付き・補助掲載）

TX-LEX の主目的は、短答で肢を見た瞬間に **条文・判例・危ない語で切れる** ようにすること。したがって、周回導線の中心は次の5項目に固定する。

| 区分 | 要素 | 位置づけ |
|---|---|---|
| **必須** | ox-grid | 肢ごとの正誤判断を先に強制する主エンジン |
| **必須** | 自己完結命題 | Lexia/SM2 で単独復習できる知識単位 |
| **必須** | 正誤 | その場で○×を確定し、誤答を潰す |
| **必須** | 文言・趣旨・射程・切断点・転用 | 誤肢切断と類似肢への転用を作る中核 |
| **必須** | 解法ナビ | 組合せ・消去・検討順を案内する。ただし知識カードには混ぜない |
| **条件付き** | 横断ミニ | 頻出混同だけを2〜3行で整理。長くしない |
| **条件付き** | 比較ミニ表 | Aなら○／Bなら×が一瞬で見える場合に限る |
| **条件付き** | 条文・判例カード | 文言・保護法益・判例射程が肢の正誤に直結する場合に置く |
| **補助掲載** | 大きいマインドマップ | 削除しないが、周回の主導線にはしない。直前確認・迷子防止用 |
| **補助掲載** | 長いストーリー解説 | 削除しないが、通常周回では読ませない。最後の確認用 |
| **補助掲載** | 深掘りコラム | 削除しないが、JX/ATHENA 側に寄せる。TX-LEX では補助に留める |
| **補助掲載** | 論文答案風の厚い説明 | 削除しないが、ARIADNE/RX の領域。TX-LEX では主教材化しない |

- TX-LEX は美しくても「読む教材」になったら負け。画面上の主役は常に ox-grid と反射コア。
- 横断・比較・条文判例カードは、短答の切断速度を上げるときだけ使う。
- マインドマップ・ストーリー・深掘り・論文答案風説明は、載せてよい。ただしページ後半・補助セクション・必要時参照の位置づけにする。
- 補助掲載要素を入れる場合も、title/code/fileName/footer/data-rx/data-athena-code に影響させない。

## 3. 350方式（組合せ型 Lexia用）の構造

各空欄①〜⑤＝1 ox-row。3要素：
1. **設問** `.ox-q`（解く）
2. **過去問選択肢** `.ox-btn`（`data-value`=letter・表示のみ・記号OK）
3. **自己完結命題** `.ox-stmt`（`.ox-core-wrap` 内・reveal 後に展開・**プールに入るのはこれだけ**）

- `data-correct-value` は letter 連結（例 `bcfgj`）。**既存採点エンジン（handleOxBtn/handleRevealAnswerBtn・value非依存）がそのまま採点**。
- verdict 表は「空欄｜正解｜論点のコア」。`data-answer-key="1:b,2:c,3:f,4:g,5:j"`（letter）で G23 充足。
- **解法ナビ**＝回答ボタンを持たず、検討順（ORDER）＋💡コツ＋組合せ消去パネルでガイド。ユーザーは下の ox-grid で逐一回答→候補が絞られる→最後に reveal。
- 配色：正解＝**淡緑地＋濃緑字**（design system）。コツ・論点のコアは**ラベル見出し＋本文を次行で字下げ（1.5em ハンギング）**。

## 3-bis. ○×方式（通常型 Lexia用・001 基準）

- ox-grid は各記述 ○/×（既に「選択肢＝○×」で自己完結）。
- 解法ナビは○×型エンジン（記述を順に判定→「正しい/誤っているものはどれか」の候補から消去→残り番号→確定で採点）。
- mode は `data-correct-value` の ○/× 個数で自動判定（○1個=正しいものはどれか／×1個=誤っているものはどれか）。

## 5. 全展開の手順（ローカルPC）

1. 各問の PART A 出題形式を判定（§1 の4分類）。
2. 公式 `000_TX/刑TXNNN.html`＝過去問の単一5択に（既存 ox-grid から or PDF から）。
3. Lexia用 `ux/000_TX/刑TXNNN_lex.html` を type 別テンプレで生成：
   - ①②→350方式（`刑TX350_lex.html` を基準に空欄→設問＋選択肢＋命題）。
   - ③→○×＋記号→実体置換（`刑TX057_lex.html` 基準）。
   - ④→素の○×＋軽量ナビ（`刑TX001_lex.html` 基準）。
4. `validate-tx-core.py`（_lex）G1〜G30 PASS／`check-duplicates.py outputs` PASS を確認。
5. commit/push（pre-commit フックで `lexia-genmeta` スタンプ）。
6. **記号のみ6問（114/119/174/218/344/351）を最優先**で350化（現状プール価値ゼロ）。

- 参照スクリプト：`scripts/lex/build-ox-lex.py`（○×型＝タイプ④の一括ビルダ雛形。既存 ox-grid から `_lex`＋公式を生成、mode は ○/× 個数で自動判定）。**注：内部 `ROOT` がリモート用 `/home/user/bar-exam` 固定＝ローカル実行時はパス調整が必要**。組合せ型(350)の変換手順は本書 §3。
- ナビ文面（💡コツ・論点のコア命題）は問題ごとに執筆が必要（量が多い＝複数セッション／バッチ）。
