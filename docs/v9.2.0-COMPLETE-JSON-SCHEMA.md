# v9.2.0 DEEP-DIVE 完全 JSON Schema 仕様書

> **作成日**: 2026-05-26
> **由来**: `scripts/render.py` 完全逆解析（build_slot_dict + render_basis + render_c1〜c7 等）
> **目的**: PDF → JSON 起草時の必須/任意フィールド完全リスト・CC night-batch-runner の入力 schema 統一

## 0. 結論サマリ

**v9.2.0 DEEP-DIVE で完全動作する JSON は top-level 28 keys + part_c 7 サブセクション**。

| カテゴリ | キー | 必須性 | 影響 |
|---|---|---|---|
| メタ | id, subject, instruction_type | **必須** | 未指定で RuntimeError or render 失敗 |
| 基本情報 | exam, year, page, crime, source, points, correct_rate, instruction | 推奨 | 未指定でヘッダー情報欠落 |
| 解答 | answer, answer_explanation | 推奨 | 未指定で解答セクション空 |
| パターン | override_pattern (P1/P2/P3), spec_version | 推奨 | 未指定で P1/v9.1.0 既定 |
| 章節 | chapter, section | 推奨 | doc-header の章/節表示に使用 |
| 事例 | case | 任意 | 事例型問題で使用 |
| 選択肢 | choices | **必須** | 各記述の本体 |
| **共通根拠** | **basis** | **推奨** | **未指定で「BASIS」セクション TODO 表示** |
| **PART C** | **part_c** | **推奨** | **未指定で C-1〜C-7 すべて空白 TODO 表示** |
| v9.2.0 拡張 | mindmap_tree, mindmap_radial, flowchart_v2, theory_deep_dive | 任意 | 未指定で該当 SVG/図表が出ない |
| ドリル | drill_blocks | 任意 | 未指定でドリル枝に空 |
| フッター | footer, palette_strategy | 任意 | footer-spec 補完 |

**根本原因**: 303-307.json は **basis / part_c / chapter / section** の 4 keys を欠いていた。これが「劣化版」の正体。

## 1. Top-Level 必須 3 keys

### 1.1 `id` (string, **必須**)
```json
{ "id": "303" }
```
- 3 桁数字文字列。`PROBLEM_ID` slot に注入され、HTML タイトル・doc-header・footer 全てに使用。

### 1.2 `subject` (string, **必須**・default="KEI")
```json
{ "subject": "KEI" }
```
- 値: `"KEI"` (刑法) / `"MIN"` (民法) / `"SYO"` (商法) / `"KEN"` (憲法) / `"GSE"` (行政) / `"KEIS"` (刑事訴訟) / `"MINS"` (民事訴訟)
- 不明値で `RuntimeError`。

### 1.3 `instruction_type` (string, **必須**)
```json
{ "instruction_type": "single-choice-5" }
```
- 値: `"single-choice-5"` / `"ox-grid-5"` / `"ox-grid-10"` / `"multi-select-5"` / `"fill-in-5"` 等
- 不明値で render_toc / render_pre_part_a / render_basis_secnav / render_part_a / render_a2 / render_part_b 全てで RuntimeError。

## 2. ヘッダー情報 (推奨)

```json
{
  "exam": "H28",
  "year": "2016",
  "page": "8",
  "crime": "横領罪の成否",
  "source": "H28-2",
  "points": "4",
  "correct_rate": "38",
  "instruction": "次のアからオまでの各記述を判例の立場に従って検討し...",
  "answer": "1",
  "answer_explanation": "正解は記述4・5...",
  "override_pattern": "P3",
  "spec_version": "v9.2.0",
  "chapter": "第6章 財産罪",
  "section": "第2節 横領罪・背任罪"
}
```

**override_pattern**: 配色パターン。P1 (易問・赤系) / P2 (中問・緑系) / P3 (難問・紫系)。

**spec_version**: `"v9.2.0"` 指定時に DEEP-DIVE 機能（tree mindmap / radial mindmap / flowchart_v2 / theory_deep_dive / professor-density-v2 / palette-derivatives）が有効化。

## 3. **`basis` (共通根拠条文・判例カード) ★必須レベル**

A-3 共通根拠条文・判例セクションを埋める。**未指定で「TODO」が表示される（303-307 の現状）**。

### Schema

```json
{
  "basis": {
    "cards": [
      {
        "kind": "statute",
        "id": "law-247",
        "icon": "📜",
        "title_html": "刑法247条 背任",
        "title_suffix_html": "",
        "freq": "high",
        "paragraphs": [
          {
            "para_num": "①",
            "body_html": "他人のためにその事務を処理する者が、自己若しくは第三者の利益を図り又は本人に損害を加える目的で、その任務に背く行為をし、本人に財産上の損害を加えたときは、五年以下の拘禁刑又は五十万円以下の罰金に処する。"
          }
        ],
        "note_html": "横領罪と背任罪は、自己が占有する他人の物に対する不法領得意思の発現が前者..."
      },
      {
        "kind": "case",
        "id": "case-saiko-s30-12-26",
        "icon": "⚖",
        "title_html": "最判昭30.12.26",
        "title_suffix_html": "（不動産二重売買・山林事件）",
        "freq": "high",
        "facts_html": "山林の二重売買の事案について、判例は...",
        "judgment_html": "本判例の理論的核心は、不動産における占有を登記名義の所在を中心に判断し...",
        "back_links": [
          { "label": "記述ア解説", "href": "#ref-case-saiko-s30-12-26-001" },
          { "label": "あてはめ", "href": "#ref-case-saiko-s30-12-26-002" }
        ]
      }
    ]
  }
}
```

### フィールド説明

| フィールド | 必須 | 説明 |
|---|---|---|
| `kind` | ★ | `"statute"` (条文) / `"case"` (判例) |
| `id` | ★ | 一意 ID。`law-247` / `case-saiko-s30-12-26` 等。**ref-case で参照される** |
| `icon` | optional | 省略時は `📜` (statute) / `⚖` (case) |
| `title_html` | ★ | カード見出し（raw HTML 可）|
| `title_suffix_html` | optional | 見出し後ろの補足（百選番号等）|
| `freq` | ★ | `"high"` (★★★) / `"mid"` (★★) / `"low"` (★) |
| `paragraphs` | statute のみ | `[{ para_num, body_html }, ...]` |
| `facts_html` | case のみ | 【事案】本文 |
| `judgment_html` | case のみ | 【判旨】本文 |
| `note_html` | optional | カード末尾の補足ノート |
| `back_links` | optional | `[{ label, href }]` — 記述本文から戻るリンク用 chip |

## 8. ref-case リンク貼り方ルール

各 `body_html` / `explanation` / `professor.points` 等の HTML 中に、判例参照は以下の形式で書く：

```html
<a class="ref-case" href="#case-saiko-s30-12-26">最判昭30.12.26</a>
```

- `href` = `#` + `basis.cards[].id`（必ず一致）
- render.py の post-process (`inject_ref_ids`) が自動で `id="ref-case-saiko-s30-12-26-001"` 等の連番 id を注入し、`basis.cards[].back_links` で逆参照可能にする。

条文の場合：
```html
<a class="ref-stat" href="#law-247">背任罪 (247条)</a>
```

これだけで `cmp-table` / `professor` / `summary_html` 等すべての本文中に**判例ネットワーク**が自動構築される。

## 9. PDF → JSON 起草プロセス（CC 自動化向け）

### Step 1: PDF テキスト抽出
- 各記述ア〜オの原文、解説、判例引用、条文引用を抽出。

### Step 2: メタ確定
- `id` / `subject` / `instruction_type` / `exam` / `correct_rate` / `crime` を確定。
- `override_pattern` を correct_rate から自動判定（推奨：>70% → P1, 40-70% → P2, <40% → P3）。

### Step 3: 判例・条文インベントリ
- PDF 中に出現するすべての判例・条文をリスト化。
- 各々に一意 `id` を割り当て（`case-saiko-s30-12-26` / `law-247` 等）。
- これが `basis.cards[]` の基礎。

### Step 4: basis 起草
- 条文: 法令データから本文取得（または PDF 引用）。
- 判例: 事案 + 判旨を PDF or 判例集から取得・新規執筆。

### Step 5: choices 起草
- 各記述ア〜オの `stem` / `original` / `explanation` を抽出。
- `professor` 4 領域（points/process/image/application）を density-v2 規律で執筆。
- 本文中の判例言及は **`<a class="ref-case">`** で必ずタグ付け。

### Step 6: part_c 起草（7 サブセクション）
- C-1: 体系的解説（趣旨 + 結論一覧テーブル）
- C-2: 概念比較テーブル
- C-3: 関連科目接続（民法/刑訴/行政等）
- C-4: 学説対立テーブル
- C-5: フローチャート（SVG 手書き or `flowchart_v2` で自動生成）
- C-6: 関連問題・出題傾向
- C-7: 三層構造記憶

### Step 7: 検証
- `python scripts/render.py {id}` で HTML 生成。
- `python scripts/validate-tx.py outputs/tx/{subject}TX/{prefix}TX{id}.html` で S1-S92 全 PASS 確認。
- 目視: ブラウザで開いて C-1〜C-7 が埋まっている / ref-case リンクが効いている / 配色が override_pattern に従っている を確認。
