# TX `_lex` ○×グリッド健全性 監査（刑TX360–445 起点・2026-07-08）

刑TX368 のユーザー指摘（「○×の付け方がわからない／解法ナビが無意味／記号の組合せの
当否を聞くな／問題文は過去問どおり verbatim に」）を起点に、TX `_lex` の ○× 判定機構を
徹底監査した記録。原因・検出ゲート・修正方針の正典。

## 1. 根本原因（root cause）

TX `_lex` は「**記述単位○×**（各記述が独立に true/false）」を単一の学習・SM2 記録面にする。
これは **5択（○×・個数）型**には自然に対応するが、**native が独立記述○×でない問題型**を
無理に流し込むと必ず破綻する：

| native の問題型 | 記述単位○×へ流し込んだ結果 |
|---|---|
| **組合せ**（①〜④に語句群を当てはめ 1〜5 から選ぶ） | 「組合せN は正しいか？○×」＝記号の当否を暗記させる無意味設問。さらに見出し/answer-key を事例単位、カード本体を組合せ単位で書くと **正誤が矛盾**（刑TX368：記述2 バッジ○・key×） |
| **穴埋め**（（　）に正しい語句を入れる） | 「各空欄に入る語句が正しいか」＝正しい語句を入れた文を並べるので **全○**（判別性ゼロの退化グリッド） |
| **会話／見解対立**（学生A・B の議論の空所補充） | 同上。正しい命題ばかりで **全○** |

**要点：組合せ・穴埋め・会話の「記号／空欄／組合せ」は SM2 に残さず、独立に真偽判定できる
実体命題（論点コア・テーゼ）へ分解する。** 分解できず全○になるなら、その問題は「記述単位○×」に
適さない＝別 UI か、正誤混在の実体命題への作り替えが要る（下記 §4 は設計判断）。

## 2. 検出ゲート（再発防止・恒久化）

`scripts/check-lex-oxgrid-integrity.py`（決定論・トークン0）。4 欠陥を機械検出：

- **L1 CONTRADICTION**：記述カード `tx-v13-verdict` バッジ ⇔ answer-key（正誤表
  `data-answer-key`／`data-correct-value`）が食い違う（368 型の矛盾）。
- **L2 COMBO-SOLVENAV**：解法ナビ STEP が「組合せN…正しいか／は正しい当てはめか」と
  組合せ当否を疑問形で問う。
- **L3 COMBO-HEADER**：解答見出しが「各組合せの○×を判定」等。
- **L4 DEGENERATE-FILL**：answer-key が全同一（全○/全×・3行以上）かつ本文が穴埋め/会話/
  組合せ由来（空欄①／【組合せ】／【語句群】／【会話】）。genuine な全×5択（マーカー無し）は除外。

**恒久化：** `scripts/check-lexia-preflight.py` の常設ステップに組込済み。移行期は `--warn-only`
（検出しても exit0）で既存バックログを潰しながら運用し、新規 corpus が 0 になったら
`--oxgrid-strict` を既定化して hard ERROR にする（CLAUDE.md の「当面 WARNING・安定後 ERROR 化」方針）。

**【2026-07-11 特殊問題監査＝三層化・生成経路配線・strict化トリガー明文化】**

  pre-commit（validate-staged・block 既定）の**三層が L1 を常時ブロック**する。全コーパス実測 検出0

## 4. 未決の設計判断（穴埋め／会話型の ○× 提示方針）

組合せ型（368）は「各事例が独立に成否を持つ」ため事例単位○×へ自然分解できた。しかし
**穴埋め・会話型は正しい語句/正しい命題ばかりで、正誤混在の○×が native に存在しない**。
提示方針は 3 案あり、ユーザーの製品判断を要する：

1. **実体命題へ作り替え（正誤混在）**：空欄に対応する論点を、正命題＋典型的誤り（対立説・
   よくある誤解）を混ぜた○×へ執筆し直す。学習効果は高いが「過去問 verbatim」から乖離＋執筆重い。
2. **確認カード化（全○のまま UI を変える）**：真偽クイズでなく「正しい当てはめを確認する」
   読み物カードと明示（○×採点を外す）。最小変更だがユーザーは全○を無意味と評価済み。
3. **v13 再生成に委ねる**：canonical 側で穴埋め/会話型の標準処理を定義し、旧版をローカル再生成。

いずれも corpus 全域（42 本）に及ぶため、方針確定後に一括対応する。
