## 付録：番号ズレの原因（2026-06-07 実データで確定）

逐語を「動画の連番（旧番号）」順に並べ、各逐語の中身が実際どの PDF（重問テキスト）かを
照合結果から復元すると、**規則的な構造**が現れる：

```
旧01→PDF01, 旧02→PDF02,
旧03〜12 → PDF11〜20,   旧13 → PDF03,
旧14〜23 → PDF21〜30,   旧24 → PDF04,
旧25〜34 → PDF31〜40,   旧35 → PDF05,
旧36〜45 → PDF41〜50,   旧46 → PDF06,
旧47〜56 → PDF51〜60,   旧57 → PDF07,
旧58〜67 → PDF61〜70,   旧68 → PDF08,
旧69〜72 → PDF71〜74,   旧77 → PDF09, 旧78 → PDF10
```

「本流10問ブロック＋基礎1問（03〜10）」が一定周期（11本ごと）で繰り返される。ここから：

1. **off-by-one ではない。** 講義動画の収録順が重問テキストの問題番号順と**そもそも別系統**
   （講座独自の教育順序）であることが主因。番号バグでは生じ得ない構造的並び替え。
2. **Whisper の取り違えではない。** 各逐語の中身は正しい（全件 PDF 事案で照合済み）。
   壊れていたのは中身ではなく「動画連番＝PDF番号」という前提だけ。
3. 「1問目が番号なし／動画タイトル1がPDF2問目から」のオフセットや「ファイル1〜4は既存・5から作成」
   は副次要因。今回**全件を中身で照合し同番号へ改名**したため吸収済み。

**結合キーは番号ではなく内容**。新規取り込みは下記プロトコルに従う（CLAUDE.md §4 取り込みプロトコル）。
