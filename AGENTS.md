# AGENTS.md instructions

私はPC初心者で、コードはほとんど書けません。
Codexは作業だけで終わらず、初心者の私が次に何をすればいいかまで日本語で丁寧に案内してください。

## 作業前

- 複数ステップになりそうな場合は、最初に簡単な作業計画を出してください。
- 危険な操作、削除、上書き、依存関係の追加、デプロイ、本番環境変更は、勝手に進めず事前に確認してください。
- Git操作のうち、このリポジトリ内の通常作業で必要になるコミットとプッシュはユーザーから包括委任済みです。作業完了後、関連ファイルだけを確認してコミットし、originへプッシュしてください。
- ただし、履歴改変、強制push、リセット、ブランチ削除、タグ操作、リリース、PR作成、デプロイ、本番環境変更は事前確認してください。

## 作業中

- コマンドを出す場合は、どこで実行するのか、何をするコマンドなのか、成功するとどう見えるのかを説明してください。
- 専門用語はできるだけ避け、必要な場合は短く説明してください。
- なるべく安全で小さい変更にしてください。

## TX / JX の生成・更新ショート指示

- ユーザーが新しいセッションで「TX355-360を生成」「JX001-010を作って」「刑TX356をアップデート」のように短く依頼した場合でも、正式依頼として扱う。
- 「生成」「新規」「作って」は、`docs/canonical-lineage.md` で active 正典を確認し、TX は最新 GENESIS-CORE 系、JX は最新 ATHENA 系から新規生成する。
- 「既存」「更新」「アップデート」「最新版に」は、古いファイルへの局所パッチではなく、active 正典に合わせた再生成・移行として扱う。
- 番号範囲は `355-360`、`355〜360`、`355から360まで` を同じ意味で読む。TX / JX や科目が特定できないときだけ、ユーザーに1点確認する。
- TX / JX の生成・更新では、構造チェックだけでなく、最新法令・判例・主要学説の確認と内容レビューを行う。完了後は関連ファイルだけをコミットし、origin へプッシュする。

## 作業後の報告形式

作業後は必ず、日本語で次の形式でまとめてください。

1. 完了したこと
2. 変更したファイル
3. 確認したこと / 実行したテスト
4. 残タスク
5. 次に私がやること
